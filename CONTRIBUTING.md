# Contributing to PyFolds

Este é o guia canônico de contribuição na raiz do repositório.

## Fluxo rápido
1. Crie uma branch a partir de `main`.
2. Faça mudanças pequenas e rastreáveis.
3. Execute validações locais (testes + checks de documentação).
4. Abra PR com contexto técnico e impactos.

## Documentação interna detalhada
- Guia de contribuição interno: [`docs/development/CONTRIBUTING.md`](docs/development/CONTRIBUTING.md)
- Hub de execução/documentação: [`docs/development/HUB_CONTROLE.md`](docs/development/HUB_CONTROLE.md)
- Processo de release: [`docs/development/release_process.md`](docs/development/release_process.md)

## Requisitos mínimos para PR
- Descrição clara da mudança.
- Evidências de validação local.
- Atualização de documentação quando aplicável.
- Referência à issue/ADR quando houver impacto arquitetural/processual.
